# S2TD Converter – Telegram **SESSION → TDATA**  
Инструмент, который преобразует файлы сессий Telegram (SESSION или SESSION+JSON) в формат TDATA, используемый клиентами Telegram Prime и аналогичными.

## ⚙️ Основные функции

| Функция | Описание |
|---------|----------|
| **SESSION / SESSION+JSON → TDATA** | Преобразует файлы сессий Telegram в файл TDATA. |
| **Многопоточность** | Параллельная обработка нескольких файлов для ускорения массового конвертирования. |
| **Настройка устройства** | Использовать и корректировать настройки устройства, применяемого при подключении к сессии. |
| **Конвертация без подключения** | Возможность преобразовать SESSION+JSON в TDATA без онлайн‑соединения с аккаунтом. |
| **Совместимость со старым лаером** | Опция сохранения совместимости со старой версией Telegram лаера или запись как новый слой. |
| **Настройки из JSON** | При наличии JSON‑файла автоматически импортируются настройки устройства. |
| **Пользовательские попытки / задержки** | Устанавливать количество повторных подключений и паузу между ними. |
| **Отключение других сессий** | По желанию отключить все остальные активные сессии аккаунта во время конвертации или переавторизации. |
| **Эмуляция устройства** | Гибкая настройка эмуляции устройства, используемого для подключения к сессии. |
| **Поддержка прокси** | Для каждой соединения использовать уникальный HTTP или SOCKS5 прокси (поддерживается многострочный список). |
| **Обработка 2FA** | Автоматический импорт 2FA из JSON‑файлов (режим JSON) либо ручной ввод (не‑JSON режим). |
| **Пропуск недействительных сессий** | Если подключение к сессии невозможно, она пропускается. |
| **Сортировка результатов** | Действительные аккаунты → «valid», недействительные → «invalid». |
| **Журналирование** | Сохраняется полный журнал операции для устранения неполадок. |
| **Резервное копирование** | Исходные файлы автоматически сохраняются перед любыми изменениями. |
